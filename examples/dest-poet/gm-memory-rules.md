# 命定之诗：外部 RPG 记忆规则段落

可加入 `agents/gm-rules.md` 或等价 GM 规则中。

## 外部 RPG 叙事记忆边界

命定之诗已经拥有自己的 state/tools，以下当前值与机械结果永远以本包为准：等级、经验、属性点、HP/MP/SP、状态效果、死亡状态、金钱、命运点数、背包、装备、技能、战斗回合、战斗 HP、任务生命周期、好感度数值、契约状态、DLC、命定核心、新闻栏目。

外部 RPG 记忆在本包中启用为长期叙事记忆层：保存旧战役历史、NPC 档案、NPC 主观记忆、世界事实证据、地点/阵营叙事变化、秘密可见性、见证者、承诺、传闻、信物来历。`quest` 与 `item` 只作为 `narrative_only` 使用：可以记“谁承诺过什么”“某物为何重要”“谁见过某物”，不得把它当作当前任务状态或背包数量。

若外部记忆与本包 state/tools 冲突，当前机械状态以本包工具为准；外部记忆只作为“过去发生/被谁知道/谁相信”的证据。

## 召回纪律

长期 NPC、旧战役人物、地点旧变故、阵营情报、承诺、秘密、传闻或玩家追问过去细节可能影响当前场景时，必须先调用：

```text
mcp_rpg_recall({ actor_id, actor_type, query, location_id?, active_quest_ids? })
```

扮演 NPC 时，`actor_id` 必须是该 NPC 的稳定实体 ID。NPC 只能使用返回 `MemoryPack` 中出现的信息；当前新主角默认不知道旧主角天光逸的私密经历，除非剧情中通过可见来源获知。

当玩家追问“当时原话、具体细节、谁在场、是否真的承诺过”或摘要不足以裁定旧因果时，若工具可见，继续调用：

```text
mcp_rpg_deep_recall(...)
mcp_rpg_get_scene(...)
```

## 写入纪律

每个入戏剧本回合由 extension 自动归档完整可见 transcript。若本轮产生重大长期事实，GM 仍应额外调用 `mcp_rpg_commit_scene` 精细写入结构化事件，尤其是：承诺、背叛、秘密揭露、阵营/地点变化、重要死亡、传闻源头、NPC 信念改变。
